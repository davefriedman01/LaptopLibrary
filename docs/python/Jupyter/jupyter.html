Tips¶

How should I startup Jupyter?¶

The unix-style terminal command

jupyter lab --ContentsManager.allow_hidden=True &>> .jupyter.log &

starts up the Jupyter Lab browser application and makes visible hidden entities in Jupyter Lab’s file browser; redirects the Jupyter server output to a hidden log file in the terminal’s current working directory; and puts the Jupyter server process in the background, freeing up the terminal for use.

How do I manage the Jupyter server as a background process in the terminal?¶

To view the live Jupyter server output, run

tail -f .jupyter.log

To quit the live Jupyter server output and return to the terminal, press Ctrl-C.

To see a list of your terminal’s live background process, run

jobs

To shutdown the Jupyter server, run

fg

to bring the process to the foreground and then press Ctrl-C to shutdown Jupyter.
