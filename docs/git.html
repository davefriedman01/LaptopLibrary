git config

  • git config How do I use command git config?

    • git config --global -l List the variable-value pairs in config file ~/.gitconfig.

    • git config --global user.name myUsername Set the commit name to myUsername.

    • git config --global user.email "myEmail@example.com" Set the commit email address to myEmail@example.com.

    • git config --global init.defaultBranch main Set the default branch name to main for new repos.

Git CL

git --version Verify that git is successfully installed on your system

Run this command when you want to update your GitHub repo:
git add . && git commit -m "your commit message" && git push
or separately:
git add .
git commit -m "your commit message"
git push

Check the status of files and folders in your local project directory (i.e., tracked or no, added or no, committed or no, pushed or no, etc.):
git status

Run this set of commands the first time you connect your local project directory to your new GitHub repo:
echo "# Your Repo Name and Description, For Example" >> README.md
git init
git add README.md
git commit -m "your commit message"
git branch -M main
git remote add origin git@github.com:<github-username>/<github-repo-name>.git
git push -u origin main

git remote Gets the local name you assigned to your remote GitHub repo when you ran git remote add... (typically, origin)
git remote get-url origin
git remote set-url origin git@github.com:<github-username>/<github-repo-name>.git This is useful when you change the name of your GitHub repo and you also need to change its name in your local configuration to maintain a connection

git clone <url>

git config -l or git config --list

git lfs version
git lfs track
git lfs ls-files
git lfs clone --depth=1 https://github.com/<github-username>/<github-repo-name>.git
